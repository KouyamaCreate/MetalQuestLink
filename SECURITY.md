# Security Policy

## Supported versions

Security fixes are applied to the latest release and the default branch. Older release branches
are not currently maintained.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use this repository's
[private vulnerability reporting form](https://github.com/KouyamaCreate/MetalQuestLink/security/advisories/new).
Repository maintainers must enable **Settings > Security > Private vulnerability reporting** when
the public repository is created. Include:

- affected version or commit;
- host macOS, Unity, OpenXR runtime, and Quest model;
- impact and realistic attack scenario;
- minimal reproduction or logs with credentials and device identifiers removed;
- any proposed mitigation.

Maintainers should acknowledge a report within 7 days, provide an initial assessment within 14
days, and coordinate disclosure after a fix or mitigation is available. These are targets, not a
paid support SLA.

Never include signing credentials, OpenAI tokens, Unity licenses, private repository access, or
personal device data in reports or test artifacts.
